Chapter 12. Class and Field Cross-Reference

This chapter lists the GMFHS, SNA topology manager, and MultiSystem Manager class names. Each class entry includes the RODM class name, the parent class, and a table of defined fields.

Class Names

GMFHS classes do not use abstract syntax notation 1 (ASN.1) object identifiers. SNA topology manager and MultiSystem Manager OSI data model classes have an ASN.1 object identifier, which is the RODM class name for the class, and a descriptive class name.

Note: RODMView enables you to enter either format.

Class Name for RODM

The ASN.1 object identifier is the name used to represent the class in the RODM data cache. An example of the ASN.1 object identifier is:

1.3.18.0.0.2085 -- logicalLink --

Do not confuse the ASN.1 object identifier with the RODM ObjectID or ClassID. The RODM ObjectID and ClassID are RODM-generated identifiers used in the MyID field maintained by RODM.

Descriptive Name

The descriptive name of each class can be any of the following:
  • Single word
  • Concatenated word string
  • Single word with a concatenated numeral

    • In the concatenated word string, the first word of the string is never capitalized, but each subsequent word or abbreviation is capitalized.

    There are two exceptions, the t2-1Node class and the ob-IAU class. This descriptive class name contains numerics and a special character. Do not confuse the t2-1Node class name with the term type 2.1 node. Even though technically related, type 2.1 node is a generic term representing several child classes of the snaNode class.

Classes for Object Creation

Objects representing resources can only be created on certain classes. Only classes on which objects can be created are listed in this chapter. All fields, including fields that are inherited by a class or defined on a class, are listed for each class.
